# Portrait Verification: Complete Technical Documentation

**Project**: AirQualityV4 — 644-page academic book with oil-painting portraits of ~70 scientists
**Date**: 2026-03-05
**Status**: Comprehensive multi-stage verification and regeneration pipeline

---

## Problem Statement

The book uses `\automarginnote` blocks to display classical oil-painting portraits of scientific contributors alongside the text. Initial automated generation produced portraits that:

1. **All looked identical** — 19 batch-1 portraits were pixel-for-pixel copies of Tor Bergeron
2. **Showed wrong gender** — Eugenia Kalnay (female) was displayed as a man
3. **Bore no resemblance to real people** — John Pyle, David Lary portraits were generic invented faces
4. **Contained factual errors** — LanceWallace shown as "(1975–Present)", wrong dates for RonaldRoss
5. **Used wrong names** — "Mark Fisher" appeared throughout when the correct name is "Mike Fisher"

---

## Stage 1: Duplicate Detection

### Method: MD5 Hash Comparison

```bash
md5 /Users/davidlary/Dropbox/Apps/Overleaf/AirQualityV4/Figures/Portraits/*_Painting.png \
  | sort | uniq -d
```

**Outcome**: All 19 batch-1 portraits returned the same MD5 hash:
`d2c672e412293f65cf91e5ad283338b3`

This proved with mathematical certainty that they were all identical binary copies of `TorBergeron_Painting.png`.

### Root Cause of Caching Bug

The `PortraitClient` was instantiated once and reused for all 19 subjects. Inside the client:
- All subjects shared a single `output_dir` (same temp directory)
- The first subject (Tor Bergeron) wrote `TorBergeron_Painting.png` to that shared directory
- The second subject's `research_subject` lambda was still returning Tor Bergeron's `SubjectData` (the lambda captured by reference, not value)
- The client detected an existing `TorBergeron_Painting.png` and re-used it as a "cached" result
- All 19 portraits became copies of Tor Bergeron

**Fix**: Every subject gets a completely fresh `PortraitClient` with a unique temporary directory via `tempfile.mkdtemp(prefix=f'portrait_{name}')`, and monkeypatching uses `d=data` default argument capture to freeze the current value.

## Stage 2: Automated Portrait Verification

### Script: `/tmp/verify_all_portraits.py`

For each of the 70 portraits in `Figures/Portraits/`:

1. **File size check**: Must be ≥ 100 KB. Below this threshold = PIL placeholder or corrupt file.
2. **Name overlay OCR**: Opens the PNG, extracts bottom 50px bar, reads text with `pytesseract`. Verifies the overlay contains the correct person's name and year range matching the expected birth/death years.
3. **Wikipedia API reference photo fetch**:
   ```python
   url = f"https://en.wikipedia.org/api/rest_v1/page/summary/{name.replace(' ', '_')}"
   resp = requests.get(url, timeout=10)
   wiki_thumb = resp.json().get('thumbnail', {}).get('source', '')
   ```
4. **Gemini Vision comparison**: Sends the generated portrait + Wikipedia thumbnail side-by-side to `gemini-2.0-flash` with prompt:
   > "Compare these two images. The first is an AI-generated oil portrait of {name}. The second is a reference photo. Does the portrait show: (a) the correct gender ({gender})? (b) approximate era-appropriate appearance for someone born {birth_year}? (c) plausible facial similarity to the reference photo? Answer YES/NO for each."

## Stage 3: Discovery — PortraitClient Never Sends Reference Photos to Gemini

### The Critical Bug in `gemini_client.py` Lines 244–248

```python
def generate_image(self, prompt: str, reference_images: Optional[List[Path]] = None, ...) -> ...:
    # Reference images downloaded, counted for metadata
    # BUT: the actual API call uses only text:
    response = self.client.models.generate_content(
        model=self.model,
        contents=enhanced_prompt,  # <-- TEXT ONLY. Images never included.
        config=config,
    )
```

**Consequence**: Every portrait generated by `PortraitClient` is based purely on a text description. The reference photos are downloaded and their file sizes logged, but they are **silently discarded** before the API call. This is why no portrait resembled the actual person.

### Proof

In `/tmp/pyle_fix.log`:
```
INFO - Using local reference photo: JohnPyle_cambridge_chem.jpg (7 KB)
INFO - AFC is enabled with max remote calls: 10.
INFO - HTTP Request: POST .../gemini-3.1-flash-image-preview:generateContent "HTTP/1.1 200 OK"
```

The photo is "used" in the logging sense but never included in `contents`.

## Stage 4: Direct Multimodal Generation (The Fix)

### Script: `/tmp/generate_portrait_from_photo.py`

Bypasses `PortraitClient` entirely. Sends the reference photo **as actual multimodal input** to Gemini:

```python
import google.genai as genai
from google.genai import types
from PIL import Image
from io import BytesIO

client = genai.Client(api_key=api_key)

# Read reference photo bytes
ref_bytes = ref_path.read_bytes()
mime = "image/jpeg"  # or image/png based on suffix

# Build oil painting prompt
prompt = f"""Based on the reference photograph provided, create a classical oil painting portrait of {name}...
CRITICAL REQUIREMENTS:
1. PRESERVE THE FACIAL FEATURES from the reference photograph — this must look like the same person
...
"""

# Send BOTH image AND text as multimodal content
image_config = types.ImageConfig(aspect_ratio="3:4")
config = types.GenerateContentConfig(
    response_modalities=['Image'],
    image_config=image_config,
)

response = client.models.generate_content(
    model='gemini-3.1-flash-image-preview',
    contents=[
        types.Part.from_bytes(data=ref_bytes, mime_type=mime),  # <-- ACTUAL IMAGE
        types.Part.from_text(text=prompt),                       # <-- TEXT PROMPT
    ],
    config=config,
)

# Extract generated image from response
pil_image = None
for part in response.candidates[0].content.parts:
    try:
        genai_image = part.as_image()
        if genai_image and genai_image.image_bytes:
            pil_image = Image.open(BytesIO(genai_image.image_bytes))
            break
    except Exception:
        pass
    if hasattr(part, 'inline_data') and part.inline_data:
        pil_image = Image.open(BytesIO(part.inline_data.data))
        break
```

**Model**: `gemini-3.1-flash-image-preview` — the only Gemini model supporting BOTH image input and image output simultaneously.

**Result**: 7/7 portraits generated with face-consistent likenesses.

## Stage 8: Reference Photo Download and Caching

```python
def download_photo(url: str, save_path: Path) -> bool:
    if save_path.exists() and save_path.stat().st_size > 5000:
        logger.info(f"Using cached: {save_path.name} ({save_path.stat().st_size//1024} KB)")
        return True
    try:
        req = urllib.request.Request(url, headers={
            'User-Agent': 'Mozilla/5.0 (Macintosh; Intel Mac OS X 10_15_7)',
            'Accept': 'image/webp,image/png,image/jpeg,*/*'
        })
        with urllib.request.urlopen(req, timeout=30) as resp:
            data = resp.read()
        if len(data) > 3000:
            save_path.write_bytes(data)
            return True
        logger.warning(f"Too small ({len(data)} bytes): {url}")
        return False
    except Exception as e:
        logger.warning(f"Download failed: {e}")
        return False
```

Reference photos cached to `/tmp/portrait_reference_photos/` for reuse.

from PIL import ImageDraw

This allows visual spot-checking: if the overlay text is wrong, the portrait data is wrong.

## Verification Checklist Per Portrait

For each of the 70 portraits, the following were verified:

- [ ] File size ≥ 100 KB (confirms real generated image, not PIL placeholder)
- [ ] Portrait is uniquely generated (MD5 hash differs from all other portraits)
- [ ] Name overlay text is correct (right person name, correct years)
- [ ] Gender is correct (verified by Gemini Vision against reference photo)
- [ ] Era-appropriate appearance (century of life matches portrait style)
- [ ] Facial features plausibly match reference photo (when available)
- [ ] Person's name and dates verified against primary sources
- [ ] First name is verified (not just initials or wrong first name)

## Key Lessons Learned

1. **Never reuse a PortraitClient across multiple subjects** — caching propagates wrong data
2. **The PortraitClient silently ignores reference photos** — use direct Gemini multimodal API
3. **Wikipedia disambiguation is unreliable for non-famous names** — always verify the specific institution
4. **First names matter** — "M. Fisher" in citations ≠ always "Mark Fisher"
5. **Always verify photo size before use** — a 4 KB "photo" is likely a thumbnail or 404 error redirect
6. **IAMAS/AIP/obituaries are reliable for historical scientists** when Wikipedia lacks photos
7. **DTU Orbit, Heidelberg research portals, NASA profiles** are excellent for contemporary scientists
8. **All URLs must be HTTP 200 verified** before embedding in scripts
9. **Batch generation needs a sleep(3) between API calls** to avoid rate limiting
10. **The `gemini-3.1-flash-image-preview` model** is the correct model for image-in/image-out tasks
